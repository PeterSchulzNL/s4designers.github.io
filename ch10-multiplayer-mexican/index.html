9b.1: Opzetten van je programma

Toen het filmpje dat hierboven staat werd opgenomen was de UI van de firebase website anders dan nu. Ook is er in de code die je van google krijgt om je project te koppelen sprake van een link naar:
"https://www.gstatic.com/firebasejs/7.19.1/firebase-app.js".
In deze link moet firebase-app.js veranderd worden in firebase.js. Het eindresultaat ziet er dan zo uit:
"https://www.gstatic.com/firebasejs/7.19.1/firebase.js".
Dit, plus een tour door de huidige UI is te vinden in het onderstaande filmpje.
Type mee met de screencast. Als je bij het stuk komt waar je code krijgt van google, kijk dan eerst even het einde van het tweede filmpje. Vanaf 4:34 wordt een essentiele stap uitgelegd. Aan het einde heb je als het goed is een werkende versie van de code uit de screencast.

9b.2: Multiplayer Mexican!

Je gaat je mexican spel multiplayer maken.

  • Koppel je mexican spel file aan firebase door de code die door google gegevens is BOVEN je eigen script tag te plaatsen.
  • Alle globale variabelen die je nu gebruikt moeten een waarde in de realtime database worden.
  • Maak voor alle waarden een functie die deze waarden in firebase kunnen wijzigen.
  • Maak voor alle variabelen die je had functies die "af gaan" op het moment dat een waarde wijzigt. Laat deze nog even leeg. Deze functies noemen "callback functies"
  • Gooi al je globale variabelen weg. Overal waar je code die variabelen wijzigde, vervang je die code door de betreffende wegschrijf functie.
  • Zorg ervoor dat als een waarde wijzigt, dat de juiste elementen op het scherm worden aangepast door de eerder gemaakte "callback functies".
  • Maak een beginscherm dat je speler naam vraagt. Op het moment dat er twee spelers in het spel zitten moet het spel kunnen starten middels een knop.
  • Je programma weet nu nog niet wie er aan de beurt is. Beide spelers zien precies hetzelfde. Verzin een manier om er toch voor te zorgen dat als de een aan de beurt is, bij de ander de knoppen grijs worden. De naam van de speler moet in iedergeval ook als globale variabele worden opgeslagen om deze te kunnen vergelijken met de speler die op het moment aan de beurt is (dat weet firebase).
  • Houd bij hoeveel iedereen moet zuipen en update dat.
  • Succes!
